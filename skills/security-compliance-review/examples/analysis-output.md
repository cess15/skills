### 🔴 Security Issues

#### Issue 1: SQL Injection in User Lookup

- **Severity**: Critical
- **Type**: Injection
- **CWE**: CWE-89
- **File**: `api/users.py`
- **Line**: `+18`
- **Code Snippet**:
  ```python
  cursor.execute("SELECT * FROM users WHERE id = " + user_id)
  ```
- **Description**: User-controlled `user_id` parameter is concatenated directly into SQL query without sanitization or parameterization, enabling SQL injection attacks.
- **Exploit Scenario**: Attacker sends `user_id=1 OR 1=1--` to dump the entire users table, or `user_id=1; DROP TABLE users--` to delete all user data. This provides complete database access.
- **Recommendation**: Use parameterized queries to prevent SQL injection:
  ```python
  cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))
  ```

#### Issue 2: Hardcoded API Key in Source Code

- **Severity**: Critical
- **Type**: Secrets Exposure
- **CWE**: CWE-798
- **File**: `config/settings.py`
- **Line**: `+11`
- **Code Snippet**:
  ```python
  API_KEY = "sk_live_1234567890abcdefghijklmnop"
  ```
- **Description**: Production API key hardcoded in source code and committed to version control. This credential is now permanently exposed in git history.
- **Exploit Scenario**: Anyone with repository access (or if leaked) can use this API key to make unauthorized API calls, potentially causing financial damage, data breaches, or service disruption.
- **Recommendation**: 
  1. **URGENT**: Rotate this API key immediately
  2. Use environment variables:
  ```python
  API_KEY = os.environ.get('API_KEY')
  if not API_KEY:
      raise ValueError("API_KEY environment variable not set")
  ```
  3. Use secrets management (AWS Secrets Manager, HashiCorp Vault, etc.)

#### Issue 3: Removed Authentication Check on Admin Endpoint

- **Severity**: Critical
- **Type**: Broken Authentication
- **CWE**: CWE-306
- **File**: `middleware/auth.py`
- **Line**: `-23 to -24` (removed lines)
- **Code Snippet**:
  ```python
  # Removed authentication:
  # if not check_admin_access(session.get('user_id')):
  #     return jsonify({'error': 'Unauthorized'}), 403
  ```
- **Description**: Authentication check was removed from admin panel endpoint, allowing unauthenticated access to sensitive administrative functions.
- **Exploit Scenario**: Any unauthenticated user can access `/admin/users` endpoint to view all user data, potentially including PII, without any credentials. This is a complete authorization bypass.
- **Recommendation**: Restore the authentication check immediately:
  ```python
  @app.route('/admin/users')
  def admin_panel():
      if not check_admin_access(session.get('user_id')):
          return jsonify({'error': 'Unauthorized'}), 403
      users = User.query.all()
      return render_template('admin.html', users=users)
  ```

#### Issue 4: Sensitive Data Logging (PII Exposure)

- **Severity**: High
- **Type**: Sensitive Data Exposure
- **CWE**: CWE-532
- **File**: `utils/logger.py`
- **Line**: `+13`
- **Code Snippet**:
  ```python
  logger.info(
      f"User {user.email} with password {user.password} performed action: {action}"
  )
  ```
- **Description**: User email and plaintext password are being logged, exposing credentials and PII in application logs.
- **Exploit Scenario**: Anyone with log access (developers, ops team, attackers who compromise the system) can harvest user credentials. Logs are often stored long-term and may be backed up to insecure locations.
- **Recommendation**: Never log passwords or PII:
  ```python
  logger.info(f"User {user.id} performed action: {action}")
  ```

#### Issue 5: Debug Mode Enabled in Production

- **Severity**: High
- **Type**: Security Misconfiguration
- **CWE**: CWE-489
- **File**: `config/settings.py`
- **Line**: `+8`
- **Code Snippet**:
  ```python
  DEBUG = True
  ```
- **Description**: Debug mode exposes stack traces, environment variables, and internal application structure to end users.
- **Exploit Scenario**: When an error occurs, attackers receive detailed stack traces revealing file paths, library versions, database connection strings, and potentially sensitive environment variables. This information aids in crafting targeted exploits.
- **Recommendation**: Always disable debug mode in production:
  ```python
  DEBUG = os.environ.get('DEBUG', 'False').lower() == 'true'
  ```
